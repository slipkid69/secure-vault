5. Security Measures in Full Detail

Duress Phrase Lockout: A system recognises phrases spoken naturally by the authorised user that sound innocuous to others but trigger a total lockout. These can be selected by the user beforehand and stored via secure hash.

System Failure Illusion: When activated, the system goes dark. It returns innocuous errors like "system failure" or "timeout," giving no indication a lockout was triggered.

Progressive Invisibility: Repeated access attempts after lockout yield complete silence—no system response. Even pings or validation checks are ignored.

Physical Token Unlock: If locked out, reentry may only occur via a token stored in a secure vault (USB or other form), only accessible to the authorised individual or organisation.

Posthumous Unlock Protocol: In case the user is deceased or unreachable, a will or timed unlock protocol can be initiated, opening access only to trusted agents after a verification window.

Puzzle Unlock Variant: An optional memory-based challenge unique to the user can be presented to regain access. These challenges must be memorised in advance and would be impossible to guess.

Cooldown Confirmation Step: When a duress signal is nearly triggered, the user may be prompted with an invisible internal confirmation mechanism. If unconfirmed within seconds, the lockout activates silently.

Signal Dispatch on Lockout: Once lockout is confirmed, a silent alert is sent to headquarters or the home node, indicating the user may be under threat. This dispatch contains no sensitive data, only a duress code or timestamp of shutdown.
